# CLAUDE.md

ビジネスパーソン向けビジネス知識学習教材プロジェクト。

## 詳細情報

プロジェクトの全体設計は [README.md](README.md) を参照。

## コンテンツ作成ガイドライン

### 教育設計

- **導入問題から始める**: 日常の疑問や考えさせる問いから入る
  - 例：「今日の100円と明日の100円、どちらが価値がある？」→ 正味現在価値
  - 例：「マネージャーに必要な要素とは？」→ ドラッカーの真摯さ
- 問いかけ → 考えるポイント → 解説という流れを守る

### フォーマット

`docs/TEMPLATE.md` に従って記事を作成する。

### Mermaid図

フレームワーク解説時は積極的にMermaid図を活用する。

- 4象限マトリクス（SWOT、BCGマトリクス等）
- フロー図（バリューチェーン、プロセス等）
- 関係図（ステークホルダー、組織構造等）

### 命名規則

- ファイル名は日本語（例：`正味現在価値.md`、`SWOT分析.md`）
- ディレクトリは既存構造に従う

## 技術的注意点

### textlint - 文章校正

textlintが導入されています。記事作成時は以下に注意してください。

#### 自動チェック項目

- **文長**: 1文150文字以内（目安）
- **助詞の連続**: 「〜が〜が」などの二重使用
- **冗長表現**: 「することができます」→「できます」
- **弱い表現**: 「〜かもしれない」の多用
- **表記ゆれ**: ビジネス用語や略語の統一

#### 表記ゆれルール（prh-rules.yml）

以下の表記に自動統一されます：

**ビジネス用語（カタカナ表記に統一）**
- `business` → `ビジネス`
- `framework` → `フレームワーク`
- `leadership` → `リーダーシップ`
- `management` → `マネジメント`
- `marketing` → `マーケティング`

**略語（大文字表記に統一）**
- `roi` → `ROI`
- `npv` → `NPV`
- `kpi` → `KPI`
- `swot分析` → `SWOT分析`

新しいビジネス用語を追加する場合は `prh-rules.yml` を更新してください。

#### Mermaid図の除外

Mermaid図内のテキストをチェック対象外にする場合：

```markdown
<!-- textlint-disable -->
\`\`\`mermaid
graph TD
    A[要素1] --> B[要素2]
\`\`\`
<!-- textlint-enable -->
```

#### ローカルでのチェック

コミット前に自動チェックされますが、手動でも実行可能：

```bash
# チェックのみ
npm run lint

# 自動修正
npm run lint:fix
```
